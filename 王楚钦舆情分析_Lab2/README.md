# 王楚钦赛后微博舆情：Lab 2 情感与观点分析

本目录承接 Lab 1 的 8 场重要单打比赛样本，分析比赛结束后 24 小时内的 45 条微博正文与 61 条一级评论。研究比较 4 场胜利和 4 场失利后的舆情，但只描述所选案例，不推断微博总体舆情。

## 主要结果

- 微博正文：胜组事件均值情感分为 `+0.605`，负组为 `-0.008`；胜组正面占 `83.3%`，负组中性占 `58.3%`。
- 一级评论：胜组事件均值情感分为 `+0.427`，负组为 `-0.160`；负组仍有 `20.0%` 正面、`40.0%` 中性和 `40.0%` 负面表达。
- 负组评论中，支持鼓励占 `16.7%`，失望惋惜占 `13.3%`，批评质疑占 `16.7%`。这说明“输球”并不等于“负面舆情”，支持、惋惜与批评会同时存在。
- 模型识别了体育语境中的黑称、器材事件和排名反讽，例如“骰子”“球拍被踩断”和“世一也就是出勤率”。

以上比例均按 `event_id` 先分场计算，再对符合覆盖要求的场次取均值，不是将全部文本直接混合计算。

## 方法设计

情感 agent 通过 OpenAI 兼容接口调用 `deepseek-v4-pro`，关闭思考并优先使用 JSON mode。`03_说明与配置/sentiment_taxonomy.json` 是模型、9 类领域情绪、12 类议题及提示词的唯一配置源。

每条文本输出极性、主导/次要情绪、强度、混合标志、议题、观点摘要、置信度和理由。`sentiment_score` 不由模型直接生成，而按“极性符号 × 强度 / 5”确定性计算。提示词明确禁止依据比赛胜负推断情感极性。

聚合遵循 Lab 1 的数据契约：

1. 微博正文与一级评论按 `content_type` 分开分析；
2. 先按 `event_id` 计算场次指标，再对胜/负组取场次均值；
3. 单场少于 3 条的来源标记为稀疏并排除出该来源的胜负汇总；
4. 正文/评论的胜组和负组分别选取头条、各 Top 议题及正面/负面声音；候选优先满足 `confidence >= 0.6`，再按点赞数、情绪强度和 ID 确定性选取；
5. 体育舆情与商品评论从 9 个维度比较，并以本样本数据支撑差异叙事。

更完整的口径见 `03_说明与配置/分析方法说明.md`。

## 复现

以下命令均在仓库根目录运行。Python 3.11 已验证。

```powershell
uv venv --python 3.11 .venv
uv pip install --python .venv\Scripts\python.exe -r 王楚钦舆情分析_Lab2\requirements.txt
```

API key 优先从环境变量读取；也可放在仓库根目录的 `api.txt`。该文件已被 `.gitignore` 排除。

```powershell
$env:DEEPSEEK_API_KEY = "你的 API key"
.\.venv\Scripts\python.exe 王楚钦舆情分析_Lab2\04_复现脚本\sentiment_agent.py --probe
.\.venv\Scripts\python.exe 王楚钦舆情分析_Lab2\04_复现脚本\sentiment_agent.py
```

聚合、绘图及不联网的 Lab 3 摄入干跑：

```powershell
.\.venv\Scripts\python.exe 王楚钦舆情分析_Lab2\04_复现脚本\analyze.py
.\.venv\Scripts\python.exe 王楚钦舆情分析_Lab2\04_复现脚本\visualize.py
.\.venv\Scripts\python.exe 王楚钦舆情分析_Lab2\04_复现脚本\validate_lab3.py
```

标注过程每条成功即追加 checkpoint，程序中断后直接重跑可续传；`--redo` 会清空 checkpoint 并重新调用 API，请谨慎使用。还可用 `--limit N`、`--posts-only` 或 `--comments-only` 做小规模测试。

## 已知局限

- 样本来自移动端历史搜索与热门评论接口，不是微博完整档案，不能外推总体比例。
- 新加坡输球事件只有 1 条评论，已排除出评论胜负汇总；中国大满贯赢球事件没有可用评论，评论逐条文件中不存在该场。两项会在 Lab 3 干跑中保留为覆盖警告。
- 106 条文本全部由单一 LLM 标注。5 条人工抽检用于验证典型边界，但不能替代更大规模的双人独立编码与一致性检验。
- 不同赛事级别、轮次、对手和采样量仍可能影响胜负组差异；本结果适合作为课程流水线演示和探索性案例分析。

## 课程要求对照

Lab 2 要求的文本情感分类由三元极性和 9 类体育领域情绪完成；主要议题由 12 类 taxonomy 与胜负分组统计完成；代表性观点按正文/评论及胜/负组输出；领域差异由体育舆情与商品评论的 9 维对比完成。`sentiment_report.json`、逐条 JSONL 及 `validate_lab3.py` 共同提供与 Lab 3 的统一数据格式和集成证据。
