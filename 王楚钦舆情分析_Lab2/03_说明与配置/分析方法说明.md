# Lab 2：王楚钦赛后微博情感与观点分析方法

## 研究问题

比较王楚钦在 8 场重要单打比赛获胜和失利后 24 小时内的微博舆情，并回答三个问题：情感如何变化、用户主要关注什么、哪些观点具有代表性。项目只描述所选比赛案例，不外推微博总体舆情。

## 输入与分析单位

输入为 Lab 1 推荐的均衡数据：45 条微博正文（胜 23、负 22）和 61 条一级评论（胜 30、负 31）。正文与评论保留统一的 `event_id`、`match_result`、`hours_after_event` 和 `content_type`，但始终分开统计。

比较单位是比赛事件而非单条微博。程序先计算每场指标，再对胜组或负组中的有效场次取等权均值，避免某一场样本多或热度高而主导结论。某来源在单场少于 3 条时标记 `thin` 并排除出该来源的胜负汇总。

## 领域 taxonomy

通用正面/负面/中性不足以描述体育舆情，因此在保留三元极性的同时设置 9 类主导情绪：支持鼓励、自豪骄傲、失望惋惜、批评质疑、愤怒不满、焦虑担忧、调侃戏谑、理性分析和中性陈述。

议题分为 12 类：赛果、心理心态、技战术、对手、器材场地、裁判判罚、舆论环境、伤病身体、历史纪录、未来展望、爱国民族和其他。完整定义及提示词以 `sentiment_taxonomy.json` 为唯一配置源。

## Agent 标注与质量控制

`sentiment_agent.py` 通过 OpenAI 兼容 API 调用 `deepseek-v4-pro`。请求优先关闭思考并开启 JSON mode；probe 会从“关闭思考 + JSON mode”开始按 4 级配置回退。API transport 与结构校验分别重试，单条失败只写失败占位，不中断整批。

模型对每条文本输出主导情绪、次要情绪、混合标志、极性、1–5 强度、1–3 个议题、观点摘要、置信度和理由。提示词最高优先级规则是“比赛胜负不等于情感极性”。乱码、纯表情或语义不足的文本要求置信度不高于 0.3。

模型不直接输出连续情感分。程序按下式确定性派生：

`sentiment_score = polarity_sign × sentiment_intensity / 5`

其中 positive、neutral、negative 的符号分别为 `+1`、`0`、`-1`。

## 聚合与代表性观点

单场指标包括极性占比、平均情感分、互动加权分、主导情绪分布、议题提及率、平均强度、混合率、低置信度数和 24 小时时段分数。胜负汇总对有效场次的指标取算术均值。

正文/评论中的胜组和负组分别选取代表性观点，包括组内头条、主要议题代表及正面/负面声音。候选记录原则上须满足 `confidence >= 0.6`，再依次按点赞数、情绪强度和记录 ID 排序，以保证结果可复现。

## 与商品评论的差异

报告从情感对象、正向核心、负向核心、领域类别、通用类别、评价维度、强度来源、中性表达和混合情感 9 个维度比较体育舆情与商品评论。体育文本的对象是运动员本人，常包含拟社会支持、国家身份、自豪、惜败和赛后继续支持；因此“败亦支持”和正负混合比商品差评更常见。

## Lab 3 数据契约

Lab 3 可读取逐条 JSONL 获取 `event_id`、`match_result`、`content_type`、全部 `sentiment_*` 字段、`topic_tags` 和 `viewpoint`，也可直接读取 `sentiment_report.json` 获取事件级及胜负级聚合结果。

`validate_lab3.py` 在不调用 API 的情况下检查 JSON 可解析性、字段存在与类型、枚举范围、事件映射、24 小时时窗、情感分公式、ID 唯一性、聚合报告计数及 5 条人工抽检回归。零记录与少于 3 条的事件记为覆盖警告，不伪装成格式错误。

## 局限

历史搜索和热门评论不是完整档案；新加坡输球评论仅 1 条，中国大满贯赢球无可用评论。单模型标注与 5 条人工抽检不足以估计严格的分类准确率，若用于论文，应增加分层随机人工样本、双人编码和一致性指标。
